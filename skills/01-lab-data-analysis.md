# 实验室数据分析技能

## 触发条件
当用户提到"实验室数据"、"检测数据"、"水质分析"、"超标分析"、"数据评价"时自动激活。

## 核心工作流

### 一、数据质控审查（必做第一步）
每次拿到检测数据，先进行质控审查，不要直接分析：
1. **检测方法适用性**：检查检测方法是否适用于高矿化度水样（如太阳山案例中甲亚胺-H法在高盐水中的基质抑制问题）
2. **检出限合理性**：检出限是否低于评价标准，标注"< 检出限"数据的处理方式
3. **平行样/空白样**：是否有质控数据，偏差是否在允许范围内
4. **采样与送样**：采样方法、保存条件、送检时效是否符合规范

### 二、评价标准选择
- 地下水：GB/T 14848-2017（明确使用哪一类标准：III类/IV类/V类）
- 地表水：GB 3838-2002
- 土壤：GB 36600-2018（建设用地）或 GB 15618-2018（农用地）
- 污水排放：对照行业排放标准或 GB 8978
- **必须在报告中明确标注评价依据和选用标准类别的理由**

### 三、数据分析框架
1. **统计描述**：最大值、最小值、均值、中位数、超标率、超标倍数
2. **空间分析**：按责任边界（企业内/企业外/园区外）分区统计
3. **时序分析**：按监测轮次对比浓度变化趋势（上升/下降/波动/稳定）
4. **多指标关联**：硼-硫酸盐-氯化物-TDS等是否同步升高（判断共同来源）
5. **异常值识别**：标注浓度跃升点位，分析跃升原因

### 四、常见逻辑陷阱（必须警惕）
- **末端采样谬误**：不能用污水厂出水浓度否定管网中途渗漏的可能性
- **单次采样代表性**：一次检测不代表历史排放状况
- **检测方法切换**：不同方法的系统性偏差不能直接与浓度变化混为一谈
- **稀释效应**：混合采样会掩盖局部高浓度
- **背景值滥用**：没有对照井数据时不能断言"自然背景主导"
