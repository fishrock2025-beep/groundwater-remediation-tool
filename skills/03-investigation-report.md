# 调查分析报告写作技能

## 触发条件
当用户提到"调查报告"、"分析报告"、"原因分析"、"超标原因"时自动激活。

## 报告标准结构

### 地下水污染调查报告框架
```
一、项目概况
   1.1 项目背景与任务来源
   1.2 调查目的与范围
   1.3 评价依据与标准
   
二、区域概况
   2.1 自然地理条件（气候、地形地貌）
   2.2 区域地质条件（地层岩性、构造）
   2.3 水文地质条件（含水层、地下水流向、补径排条件）
   2.4 园区/场地概况（企业分布、生产历史）

三、调查方案与实施
   3.1 监测井布设依据
   3.2 采样方法与质控措施
   3.3 检测项目与分析方法
   
四、地下水质量现状评价
   4.1 常规指标评价（分区、分层统计）
   4.2 特征污染物评价
   4.3 空间分布特征
   4.4 时序变化特征
   
五、超标原因分析（核心章节）
   5.1 天然地质背景分析
   5.2 人为污染源识别
   5.3 污染迁移途径分析
   5.4 各因素贡献判断
   
六、结论与建议
   6.1 主要结论
   6.2 下一步工作建议
```

## 写作质量标准

### 必须做到
- 数据先行：每个判断必须引用具体的监测数据
- 空间对应：文字描述与图件（平面图、剖面图）一一对应
- 逻辑闭环：提出的每一个假说都要有验证或待验证的说明
- 责任边界清晰：企业内/企业外/园区外分别统计和分析
- 对照参考：必须有背景对照点的数据对比

### 绝对不能
- 没有对照数据就下"天然背景主导"的结论
- 用污水厂末端数据否定管网渗漏可能
- 把检测方法差异导致的数据偏差全部归因于浓度真实变化
- 结论部分出现"不排除""可能存在"等模糊措辞超过2处
- 只列数据不分析，或只有分析没有数据支撑

### 审核报告时的检查清单
1. 评价标准选用是否正确？理由是否充分？
2. 数据质控是否完整？异常数据是否说明原因？
3. 空间分析是否覆盖所有责任边界？
4. 时序分析是否涵盖所有可用轮次数据？
5. 原因分析是否考虑了所有可能的来源（天然+人为）？
6. 结论是否与数据分析一致（不夸大、不缩小）？
7. 建议是否具有可操作性？是否包含断源措施？
